# 任务完成与归档协议

## 适用范围
本协议适用于 **XH-202630 科研文献智能助手** 项目（Veritas/求真）的所有功能开发任务，每次功能开发完成后必须严格执行。

---

## 执行时机
**触发条件**：当用户说以下任意话语时，立即执行本协议：
- "完成了"
- "做完了"
- "开发完成"
- "功能已实现"
- "测试通过"
- 类似表示任务结束的语句

---

## 归档任务清单（必须全部完成）

### 0. 确定归档模块
- 判断本次开发任务属于哪个模块：AI服务（ai-service）、Java后端（backend）或前端（frontend）
- 归档目标目录为 `log/{模块名}/`，例如 `log/ai-service/`

### 1. 确定编号
- 扫描对应模块子目录（如 `log/ai-service/`）下所有 `数字编号_` 格式的文件夹
- 找出当前最大编号（例如：`03_RAG混合检索`）
- 新编号 = 最大编号 + 1（例如：`04_`）
- 若该模块子目录下尚无编号文件夹，则从 `01` 开始

### 2. 创建归档文件夹
- **位置**：`log/` 目录下对应模块子目录中，即：
  - AI服务开发：`Veritas(求真)/log/ai-service/`
  - 后端开发：`Veritas(求真)/log/backend/`
  - 前端开发：`Veritas(求真)/log/frontend/`
- **命名格式**：`数字编号_开发内容简述`
- **示例**：`04_RAG混合检索开发`（ai-service模块下）

### 3. 编写日志文档
在归档文件夹内创建 `README.md`，必须包含以下内容：

```markdown
# 功能名称

## 功能描述
- 解决了什么问题
- 实现了什么功能
- 业务价值

## 实现逻辑
- 修改的核心文件列表
- 使用的算法或设计模式
- 关键代码逻辑说明

## 接口变更
### Request
```json
{
  // 请求示例
}
```

### Response
```json
{
  // 响应示例
}
```

## 测试结果
- 测试场景1：结果
- 测试场景2：结果
- 是否通过：是/否

## 相关文件
- 涉及的代码文件路径
- 配置文件变更
```

### 4. 创建教学文档
在归档文件夹内创建 `TEACH.md`，用于记录本次开发的技术教学和经验总结，必须包含以下内容：

```markdown
# 技术教学文档

## 开发思路
- 需求分析过程
- 技术选型考虑
- 架构设计思路
- 遇到的问题及解决方案

## 实现步骤
1. 第一步：...
2. 第二步：...
3. 第三步：...

## 解决了什么问题
- 核心问题描述
- 解决方案对比
- 最终方案的优势

## 变更内容
### 新增文件
- 文件路径及作用

### 修改文件
- 文件路径及变更点

### 配置变更
- 配置项及说明

## 关键技术点
- 使用的核心技术
- 代码实现亮点
- 需要注意的细节

## 经验总结
- 开发过程中的收获
- 踩过的坑及如何避免
- 最佳实践建议
```

### 5. 整理过程产物
将以下文件移入归档文件夹：
- 临时文档
- 测试数据
- 截图/录屏
- 其他开发过程中产生的文件

### 6. 确认归档完成
- [ ] 归档模块判断正确（ai-service / backend / frontend）
- [ ] 归档文件夹已创建在正确的模块子目录下
- [ ] 文件夹命名正确（编号连续）
- [ ] README.md 已编写且内容完整
- [ ] TEACH.md 已编写且内容完整
- [ ] 所有过程产物已移入文件夹
- [ ] 根目录及模块源码目录无遗留临时文件

## 物理结构约束

### 禁止事项
- ❌ 在 log/ 文件夹外随意创建归档文件夹
- ❌ 将归档文件夹放在错误的模块子目录下（如将前端日志放入 backend/）
- ❌ 直接放在 log/ 根目录而不放入对应模块子目录
- ❌ 使用无编号或随意命名的文件夹
- ❌ 遗漏README.md文件
- ❌ 遗漏TEACH.md文件
- ❌ 在根目录堆积开发过程文件

## 完成标准

**只有在满足以下所有条件后，才算任务正式结束：**

1. ✅ 归档文件夹已创建并命名正确
2. ✅ README.md 已编写并包含所有必需内容
3. ✅ TEACH.md 已编写并包含所有必需内容（开发思路、实现步骤、解决问题、变更内容）
4. ✅ 所有过程产物已整理到归档文件夹
5. ✅ 已向用户确认归档完成

---

## 注意事项

1. **严禁跳过**：本协议为强制性要求，不得以任何理由跳过
2. **立即执行**：任务完成后应立即执行归档，不得延迟
3. **完整性**：README.md 和 TEACH.md 内容必须详实，便于后续查阅和学习
4. **教学文档价值**：TEACH.md 是知识传承的重要载体，需认真总结技术要点和经验教训
5. **编号连续性**：确保编号连续，无跳号或重复
6. **归档位置**：所有归档文件夹统一放在 `log/{模块名}/` 目录下，按模块（ai-service / backend / frontend）分别归档，不直接放在 log/ 根目录
