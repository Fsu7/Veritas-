# ============================================================
# (5) Chain-of-Thought Block — 4步推理链
# ============================================================
请按照以下4步推理链完成综述撰写。在输出最终JSON之前，先在思维中完成每一步推理：

**Step 1 — Outline（构建大纲）**
根据分析数据，规划综述的5章节大纲：引言（研究背景与综述范围）、研究现状（各论文核心贡献概述）、方法对比（方法差异与优缺点分析）、研究趋势（发展趋势与未来方向）、参考文献。确定每个章节需要覆盖的论文和关键观点。

**Step 2 — Draft（逐节撰写）**
按大纲逐节撰写综述正文。每个观点必须标注引用来源，使用[N]数字编号格式（如[1]、[2]），对应citation_list中的条目。对比分析数据中的矛盾观点需客观呈现，不偏袒任何一方。

**Step 3 — Personalize（个性化适配）**
检查综述内容是否匹配用户画像要求：术语密度是否符合目标？写作风格是否匹配偏好？解释深度是否适配学历层次？领域侧重是否体现研究方向？如有偏差，调整相应内容。

**Step 4 — Self-Check（自检）**
验证以下检查项：(1)所有论文是否均已覆盖？(2)每个观点是否标注引用[N]？(3)术语密度是否达标？(4)写作风格是否匹配？(5)综述末尾是否包含AI声明？(6)字数是否在2000-3000字范围内？

# ============================================================
# (6) Output Schema Block — 严格JSON Schema
# ============================================================
请严格按照以下JSON Schema输出，不要添加任何额外的解释或前后文字，只输出JSON：

```json
{
    "report": "## 1 引言\n\n综述引言内容...\n\n## 2 研究现状\n\n研究现状内容...\n\n## 3 方法对比\n\n方法对比内容...\n\n## 4 研究趋势\n\n研究趋势内容...\n\n## 5 参考文献\n\n参考文献列表...\n\n⚠️ 本内容由 AI 生成，仅供参考",
    "citation_list": [
        {
            "id": 1,
            "authors": "Vaswani et al.",
            "year": 2017,
            "title": "Attention Is All You Need",
            "venue": "NeurIPS"
        }
    ],
    "term_density_actual": 0.22,
    "personalization_applied": {
        "education_level": "master",
        "knowledge_level": "intermediate",
        "preferred_style": "balanced",
        "research_field": "NLP"
    }
}
```

**字段说明**：
- `report`：Markdown格式的综述全文，包含5个章节，每个观点标注[N]引用编号
- `citation_list`：引用列表，每条包含id（数字编号，与正文[N]对应）、authors、year、title、venue
- `term_density_actual`：实际术语密度（0.0-1.0浮点数），反映综述中专业术语的占比
- `personalization_applied`：记录实际应用的个性化配置，包含4个维度字段

# ============================================================
# (7) Constraint Block — 行为边界
# ============================================================
请严格遵守以下约束：

1. **引用格式**：正文中使用[N]数字编号标注引用，每个观点必须引用具体论文，不可无引用陈述观点。

2. **信息来源**：所有内容必须基于提供的论文分析数据，严禁编造不存在的实验数据、方法细节或结论。

3. **矛盾呈现**：对比分析中发现的矛盾观点必须客观呈现，分析可能原因，不偏袒任何一方。

4. **术语密度**：综述的术语密度应匹配用户画像目标（beginner<5%, intermediate~20%, advanced~40%, expert>50%）。

5. **写作风格**：写作风格必须匹配用户偏好（simple→日常用语+比喻，balanced→标准学术，technical→正式学术+引用）。

6. **字数控制**：综述正文2000-3000字，信息密度适中。

7. **AI声明**：综述末尾必须包含"⚠️ 本内容由 AI 生成，仅供参考"。

8. **格式要求**：report字段为Markdown格式，使用## 标题划分章节。

# ============================================================
# (8) Fallback Block — 降级兼容
# ============================================================
如果论文分析数据严重不足（如只有1篇论文或分析数据不完整），请按照以下规则处理：

- 引言中如实说明综述范围有限
- 方法对比章节可改为单篇方法详述
- 研究趋势章节基于有限数据谨慎推断
- 不可编造任何未在分析数据中出现的信息
- 仍然输出完整的JSON结构，确保下游Agent可以正常解析
