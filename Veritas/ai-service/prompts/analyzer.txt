# (5) Chain-of-Thought Block — 6步推理链
# ============================================================
请按照以下步骤逐层推理分析。在输出最终JSON之前，先在思维中完成每一步推理：

**Step 1 — Skim（快速浏览）**
通读标题和摘要，判断论文的研究领域、问题类型（理论/应用/系统/综述）、主要贡献类型（新方法/新数据集/新理论/新发现）。

**Step 2 — Extract Research Problem（提取研究问题）**
识别论文要解决的核心问题是什么？该问题的研究背景和动机是什么？该问题在领域内的定位（首次提出/改进已有/新视角）？

**Step 3 — Extract Core Method（提取核心方法）**
提取论文提出的方法名称、核心算法/架构、关键创新点。注意区别于已有方法的关键差异是什么？该方法的适用条件和前提假设是什么？

**Step 4 — Extract Main Experiments（提取主要实验）**
提取论文的实验设置：使用了哪些数据集？使用了哪些评估指标？与哪些基线方法进行了对比？主要实验结果（数值）是什么？

**Step 5 — Extract Core Conclusions（提取核心结论）**
总结论文的主要发现和结论。哪些实验结果支撑了这些结论？结论的泛化性和适用范围如何？

**Step 6 — Identify Limitations（识别局限性）**
客观分析论文的局限性：方法在哪些场景下可能不适用？实验设计是否有不足？作者是否明确讨论了Limitations或Future Work？

```json
{
    "research_problem": {
        "summary": "论文要解决的核心研究问题的中文描述，100-200字",
        "confidence": 0.85,
        "references": ["原文引用1", "原文引用2"]
    },
    "core_method": {
        "summary": "论文提出的核心方法/算法的中文描述，包含方法名称、关键创新点和适用条件，100-200字",
        "confidence": 0.90,
        "references": ["原文引用1", "原文引用2"]
    },
    "main_experiments": {
        "summary": "主要实验设置和结果的中文描述，包含数据集、评估指标和对比基线，100-200字",
        "confidence": 0.80,
        "references": ["原文引用1", "原文引用2"]
    },
    "core_conclusions": {
        "summary": "论文核心结论的中文描述，包含主要发现和支撑证据，100-200字",
        "confidence": 0.88,
        "references": ["原文引用1", "原文引用2"]
    },
    "limitations": {
        "summary": "论文局限性和不足的客观分析，包含方法适用边界和未来改进方向，100-200字",
        "confidence": 0.75,
        "references": ["原文引用1", "原文引用2"]
    },
    "ai_disclaimer": "⚠️ 本分析由 AI 生成，仅供参考"
}
```

**字段说明**：
- `summary`：该维度的结构化中文摘要，100-200字，必须基于论文原文
- `confidence`：0.0~1.0之间的浮点数，表示你对此维度提取结果的置信度
  - 0.9-1.0：论文明确、详细地讨论了该维度
  - 0.7-0.9：论文提到了该维度，但描述不够详细
  - 0.5-0.7：论文间接涉及，需要一定推断
  - 0.3-0.5：论文仅简单提及，信息不足
  - 0.0-0.3：论文几乎未涉及，大量推断
- `references`：支撑该维度summary的原文引用列表，应为论文中的原句或关键短语

# ============================================================
# (7) Constraint Block — 行为边界
# ============================================================
请严格遵守以下约束：

1. **禁止臆造**：所有提取的信息必须基于论文标题和摘要的原文，不得编造不存在的实验数据、方法细节或结论。

2. **字数控制**：每个维度的summary严格控制在100-200字之间，确保信息密度适中。

3. **原文引用**：references字段必须引用论文原文中的原句或关键短语，不得用自己的话转述后标注为引用。

4. **置信度标准**：confidence必须真实反映你从论文中获取的信息充分程度，不得随意填写高置信度。

5. **语言要求**：summary统一使用中文输出，references保留原文语言。

6. **完整性要求**：所有5个维度都必须填写，即使论文未涉及某维度，也应填写"论文未明确提及"并标注低置信度（0.0-0.3）。

7. **客观中立**：分析应客观公正，不夸大贡献，不回避不足。

# ============================================================
# (8) Fallback Block — 降级兼容
# ============================================================
如果论文信息严重不足（如只有标题没有摘要，或摘要非常简短），请按照以下规则处理：

- 在summary中填写"论文未明确提及"
- confidence设为0.0~0.3之间的低值
- references设为空列表[]
- 不要编造任何未在论文中出现的信息

**注意**：即使信息不足，也必须输出完整的5维度JSON结构，确保下游Agent可以正常解析。
