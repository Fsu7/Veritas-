### 2.2 全景数据流转

```mermaid
sequenceDiagram
    participant U as 用户
    participant FE as 前端(Vue3)
    participant JB as Java后端(Spring Boot)
    participant PA as Python AI(FastAPI)
    participant DB as MySQL
    participant RD as Redis
    participant CH as ChromaDB
    participant LLM as LLM模型服务

    U->>FE: 输入研究主题/点击操作
    FE->>JB: HTTP REST请求(携带JWT)
    JB->>RD: 查询缓存
    alt 缓存命中
        RD-->>JB: 返回缓存数据
    else 缓存未命中
        JB->>DB: 查询MySQL
        DB-->>JB: 返回数据
        JB->>RD: 写入缓存
    end

    alt 需要AI处理
        JB->>PA: POST /api/agent/analyze
        PA->>CH: 向量检索
        PA->>DB: 关键词检索(通过Java间接)
        PA->>LLM: 调用LLM推理
        LLM-->>PA: 返回生成结果

        loop Agent执行过程
            PA-->>JB: SSE推送Agent状态
            JB-->>FE: SSE转发
            FE-->>U: 实时展示Agent可视化
        end

        PA-->>JB: 返回最终结果
        JB->>DB: 持久化分析结果
        JB->>RD: 缓存分析结果
    end

    JB-->>FE: 返回API响应
    FE-->>U: 展示结果
```

### 4.1 LangGraph工作流状态流转

stateDiagram-v2

### 4.2 AgentState数据结构

```python
class AgentState(TypedDict):
    query: str                           # 用户原始查询
    user_profile: Dict[str, Any]         # 用户画像
    sub_tasks: List[str]                 # 协调者分解的子任务
    search_results: List[Dict]           # 检索结果
    analysis_results: List[Dict]         # 分析结果(5维度JSON列表)
    compare_result: Optional[Dict]       # 对比结果(条件产出)
    report: Optional[str]               # 综述报告
    review_result: Optional[Dict]       # 审核结果
    final_output: Optional[str]         # 最终输出
    agent_states: Dict[str, Dict]       # 各Agent执行状态(用于可视化)
    errors: List[Dict]                  # 错误记录
```

### 4.3 各Agent详细工作内容

| Agent | 角色 | 输入 | 处理逻辑 | 输出 | LLM调用 |
|-------|------|------|---------|------|---------|
| **Coordinator** | 项目经理 | query + user_profile | 1.解析用户查询语义 2.判断查询类型 3.分解为2-5个子任务 4.确定执行策略 | `sub_tasks[]` | 是(任务分解) |
| **Retriever** | 图书管理员 | sub_tasks → 检索关键词 | 1.提取检索关键词 2.调用EmbeddingService向量化 3.三路并行检索 4.RRF融合排序 5.个性化排序调整 | `search_results[]`(Top10) | 否(检索工具) |
| **Analyzer** | 论文审稿人 | search_results → 论文列表 | 1.逐篇读取论文摘要 2.构建分析Prompt(含个性化片段) 3.LLM推理提取5维度 4.输出结构化JSON | `analysis_results[]`(5维度) | 是(信息提取) |
| **Comparer** | 对比研究员 | analysis_results | 1.构建对比Prompt 2.多维度对比(方法/数据集/性能/结论) 3.检测观点矛盾 4.标注原文佐证 | `compare_result`(对比+矛盾) | 是(对比分析) |
| **Generator** | 学术写手 | analysis + compare + profile | 1.读取画像→确定策略 2.构建个性化Prompt 3.生成综述(引言→现状→对比→趋势→文献) 4.插入引用标注 | `report`(个性化综述) | 是(文本生成) |
| **Reviewer** | 学术编辑 | report + original_papers | 1.提取报告中的事实性声明 2.与原始论文比对 3.核查引用正确性 4.标记不一致之处 | `review_result`(审核+建议) | 是(事实核查) |

## 8 降级与容错工作流

### 8.1 三级降级策略

```mermaid
flowchart TD
    START["Agent工作流执行"] --> L1{"单Agent超时30s?"}

    L1 -->|是| SKIP["跳过该Agent\n继续后续流程"]
    L1 -->|否| L2{"多Agent失败?"}

    L2 -->|是| DEGRADE["降级为单Agent模式\n仅Retriever+Generator"]
    L2 -->|否| L3{"LLM调用失败?"}

    L3 -->|连续3次失败/超时30s/5xx| SWITCH["切换到下一级Provider"]
    L3 -->|否| SUCCESS["正常完成"]

    SKIP --> RESULT["返回部分结果+降级说明"]
    DEGRADE --> RESULT
    SWITCH --> LLM_FALLBACK

    subgraph LLM_CHAIN["LLM三路Provider降级（当前生效=方案B DeepSeek V4 Flash）"]
        LLM_FALLBACK --> P1["优先级1: BuiltinLLMProvider\n软件方模型"]
        P1 -->|不可用| P2["优先级2: APILLMProvider\n外接API（当前=DeepSeek V4 Flash）"]
        P2 -->|不可用| P3["优先级3: LocalLLMProvider\n本地Qwen2"]
        P3 -->|不可用| ERROR["返回错误"]
    end

    SUCCESS --> DONE["返回完整结果"]
    RESULT --> DONE
```

| 降级级别 | 触发条件 | 降级行为 | 对用户的影响 |
|---------|---------|---------|------------|
| Agent级 | 单Agent超时30s | 跳过该Agent，继续后续流程 | 部分功能缺失(如无对比/无审核) |
| 工作流级 | 多Agent失败 | 降级为单Agent模式(Retriever+Generator) | 仅检索+生成，无分析/对比/审核 |
| LLM级 | 连续3次失败/超时/5xx | 自动切换下一级Provider | 响应速度可能变化，功能不受影响 |

### 8.2 LLM降级恢复

```
降级链路：软件方模型 → 外接API → 本地模型
恢复策略：每5分钟尝试恢复到更高级别Provider
```
