**核心逻辑**：
- `PythonAIClient`：基于Spring WebClient的HTTP客户端
  - 连接池配置：最大连接数20，超时30s

  - 目标地址：`http://ai-service:8000`（从application.yml读取）
- 请求/响应转换：
  ```
  Java AgentRequestDTO         →  Python AnalyzeRequest
  ├── topic: String            →  topic: str
  ├── paperIds: List<String>   →  paper_ids: List[str]
  ├── userId: String           →  user_id: str
  └── profileContext: Map      →  user_profile: UserProfile

       ├── knowledgeLevel      →  knowledge_level: str
       └── preferredStyle      →  preferred_style: str
  ```

|------|---------|--------|------------|-----|------|
| F2.6.1 | 用户画像缓存 | P0 | `user:profile:{userId}` | 1小时 | 画像低频变更，长TTL |

**核心逻辑**：
- **Cache-Aside模式**：
  ```
  读取: Redis → 命中返回 → 未命中查MySQL → 写入Redis → 返回
  写入: 写MySQL → 删除Redis缓存（而非更新）
  ```
- 10个Redis缓存空间（含论文详情、会话状态、JWT黑名单、AI降级状态等）
- Spring Cache + 自定义CacheManager统一管理

**LangGraph工作流状态定义**：

```python
class AgentState(TypedDict):
    query: str                           # 用户查询
    user_profile: Dict[str, Any]         # 用户画像
    sub_tasks: List[str]                 # 协调者分解的子任务
    search_results: List[Dict]           # 检索结果
    analysis_results: List[Dict]         # 分析结果（5维度JSON列表）
    compare_result: Optional[Dict]        # 对比结果（条件产出）
    report: Optional[str]                # 综述报告
    review_result: Optional[Dict]        # 审核结果
    final_output: Optional[str]          # 最终输出
    agent_states: Dict[str, Dict]        # 各Agent执行状态
    errors: List[Dict]                   # 错误记录
```

**工作流执行路径**：
```
START → Coordinator → Retriever → Analyzer → [论文数≥2? → Comparer] → Generator → Reviewer → END
                                                                    ↑                    │
                                                                    └── 审核不通过回退 ──┘
```

**三级降级策略**：

| 级别 | 触发条件 | 降级行为 |
|------|---------|---------|
| Agent级 | 单Agent超时30s | 跳过该Agent，继续后续流程 |
| 工作流级 | 多Agent失败 | 降级为单Agent模式（仅Retriever+Generator） |
| LLM级 | 连续3次调用失败/超时30s/HTTP 5xx | 自动切换到下一级Provider |

**三路RRF融合算法**（ADR-011）：

```
用户查询 → text-embedding-v4(阿里云百炼) → 1024维查询向量
                    │
        ┌───────────┼───────────┐
        ▼           ▼           ▼
   路径A:ChromaDB  路径B:MySQL   路径C:Neo4j
   语义检索Top20   全文检索Top20  图谱推理Top20
        │           │           │
        └───────────┼───────────┘
                    ▼
          RRF融合: score(d) = Σ 1/(k + rank_i), k=60
                    │
                    ▼
          个性化排序调整:
          score_final = score_rrf × 0.5
                     + field_match × 0.3
                     + popularity × 0.2
                    │
                    ▼
              Top10 检索结果 + 推荐理由
```

**ChromaDB配置**：
- Collection: `papers`
- 向量维度: 768 (text-embedding-v4)
- 相似度: cosine
- HNSW参数: M=16, construction_ef=200
- 元数据: paper_id, title, year, venue, citation_count, chunk_index, chunk_type
- 分块: 500-1000字/块，重叠50-100字

**Neo4j知识图谱**（ADR-011）：
- 节点: Paper / Method / Concept / Author
- 关系: USES / IMPROVES / RELATED_TO / AUTHORED_BY / CITES / BELONGS_TO

**LLM三级Provider降级**（ADR-003）：

```
优先级1: BuiltinLLMProvider（软件方提供的模型）
    ↓ 不可用时（连续3次失败/超时30s/HTTP 5xx）
优先级2: APILLMProvider（外部API：DeepSeek/OpenAI等）
    ↓ 不可用时
优先级3: LocalLLMProvider（本地Qwen2模型）
```

**降级恢复**：每5分钟尝试恢复到更高级别Provider。

### 4.2 通信方式详解

#### 4.2.1 前端 → Java后端（HTTP REST + SSE）

| 通信方式 | 方向 | 用途 | 协议 |
|---------|------|------|------|
| REST API | 前端→Java | 用户操作请求（登录/检索/分析等） | HTTP POST/GET/PUT/DELETE |
| SSE | Java→前端 | Agent执行状态实时推送 | HTTP SSE (text/event-stream) |

**前端请求拦截器**：Axios实例自动注入JWT Token（`Authorization: Bearer {token}`）

**SSE事件流**：
```
Java后端 → 前端（SseEmitter推送）
event: agent_state_update
data: {"agentName": "retriever", "status": "running", "progress": 0.6, "intermediateResult": "找到15篇相关论文"}
```

**前端SSE处理**：`useSSE` composable封装，自动重连（3s间隔，最多5次）

#### 4.2.2 Java后端 → Python AI服务（HTTP REST + SSE转发）

| 通信方式 | 方向 | 用途 | 协议 |
|---------|------|------|------|
| REST API | Java→Python | 分析请求（论文分析/对比/综述） | HTTP POST |
| SSE转发 | Python→Java→前端 | Agent执行状态实时推送 | HTTP SSE 双层转发 |

**关键通信接口**：

| Java调用 | Python端点 | 请求体 | 响应体 |
|---------|-----------|--------|--------|
| 论文分析 | `POST /api/agent/analyze` | AgentRequestDTO → AnalyzeRequest | AnalyzeResponse → AnalysisResponseDTO |
| 论文搜索 | `POST /api/search` | SearchRequest | SearchResponse |
| 模型状态 | `GET /api/model/status` | — | ModelStatusResponse |
| 健康检查 | `GET /health` | — | 健康状态JSON |

**SSE双层转发链路**（ADR-009）：

```
Python (FastAPI)
  StreamingResponse(generator, media_type="text/event-stream")
       │ HTTP SSE
       ▼
Java (Spring Boot)
  WebClient接收 → SseEmitter转发
       │ HTTP SSE
       ▼
前端 (Vue3)
  EventSource / useSSE composable接收 → agentStore更新 → AgentFlowChart可视化
```

### 4.3 共享数据层

#### 4.3.1 共享数据库

| 数据库 | 主要使用者 | 共享方式 |
|--------|-----------|---------|
| MySQL 8.0 | Java后端（主） | Java直接JPA操作，Python不直接访问MySQL |
| Redis 7.0 | Java后端（主）+ Python AI | Java管理缓存，Python写Agent状态 |
| ChromaDB 0.5+ | Python AI（主） | Python直接操作向量库，Java通过Python API间接使用 |
| Neo4j 5.x | Python AI（主） | Python直接操作知识图谱，Java通过Python API间接使用 |

**Redis共享Key**：

| Key模式 | 写入方 | 读取方 | TTL |
|---------|--------|--------|-----|
| `user:profile:{userId}` | Java | Java | 1h |
| `search:result:{hash}` | Java | Java | 10min |
| `analysis:result:{id}` | Java | Java | 30min |
| `agent:state:{analysisId}` | Python | Java→前端 | 5min |
| `ai:provider:status` | Python | Java | 5min |
| `auth:token:blacklist:{hash}` | Java | Java | Token有效期 |

#### 4.3.2 数据一致性策略

**Cache-Aside模式**（Java侧执行）：
```
写操作：写MySQL → 删除Redis缓存
读操作：查Redis → 命中返回 → 未命中查MySQL → 写入Redis → 返回
```

**MySQL ↔ ChromaDB同步**：
- 论文入库时：Java写MySQL → 调用Python API生成向量存入ChromaDB
- 通过`paper_id`外键关联

**SSE推送时序**：
- Python → Java → 前端，单向推送不回传

### 4.4 完整业务流程（模块间协作）

#### 4.4.1 智能检索流程

```
[前端] 用户输入"Multi-Agent协同决策"
  │ POST /api/papers/search?q=Multi-Agent
  ▼
[Java] PaperController接收 → PaperService处理
  │ 1. 检查Redis缓存 search:result:{hash}
  │ 2. 缓存未命中 → 调用Python AI
  ▼
[Java→Python] POST http://ai-service:8000/api/search
  │ {query: "Multi-Agent协同决策", top_k: 10, filters: {}}
  ▼
[Python] SearchService
  │ 1. text-embedding-v4编码 → 1024维查询向量
  │ 2. 三路并行检索：
  │    路径A: ChromaDB语义检索 → Top20
  │    路径B: MySQL FULLTEXT关键词检索 → Top20
  │    路径C: Neo4j图谱推理 → Top20
  │ 3. RRF融合排序 → Top10
  │ 4. 个性化排序调整（field_match×0.3 + popularity×0.2）
  ▼
[Python→Java] 返回SearchResponse
  │ {results: [{paper_id, title, abstract, score, year, venue}], total: 10}
  ▼
[Java] PaperService
  │ 1. 补充论文元数据（从MySQL/Redis）
  │ 2. 写入Redis缓存 search:result:{hash}（TTL 10min）
  ▼
[Java→前端] 返回PaperDTO[]
  │ {code: 200, data: {items: [...], pagination: {...}}}
  ▼
[前端] 展示论文卡片列表
```

#### 4.4.2 论文深度分析流程

```

  ▼

  │
  ├── [SSE] → Java → 前端: Coordinator running...
  ├── Coordinator: 分解为子任务
  ├── [SSE] → Java → 前端: Retriever running... (0.6)

  │ {analysis_id: "anl_001", status: "completed",
  │  report: "...", citations: [...], agent_states: {...}}
  ▼
[Java] AnalysisService
  │ 1. 持久化到MySQL analysis_results表
  │ 2. 写入Redis缓存 analysis:result:{analysisId}（TTL 30min）
  ▼
[Java→前端] 返回分析结果
  │ {code: 200, data: {analysisId: "anl_001", type: "paper_analysis", result: {...}}}
  ▼
```
