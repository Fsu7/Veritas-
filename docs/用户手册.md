## 1 欢迎使用

### 1.1 系统简介

科研文献智能助手是一个**基于多智能体协同**的科研文献分析平台。您只需要输入研究主题，系统便会自动派出六个 AI 智能体（Agent）协同工作，帮您完成：

| 功能 | 说明 |
|------|------|
| **智能文献检索** | 语义理解 + 关键词匹配 + 知识图谱推理，三路融合检索 |
| **论文深度分析** | 自动提取研究问题、核心方法、主要实验、核心结论、局限性 |
| **多论文对比** | 方法、数据集、性能、结论四维对比，自动发现观点矛盾 |
| **个性化综述生成** | 根据您的研究方向和知识水平，生成差异化综述报告 |
| **Agent协同可视化** | 实时观看6个AI智能体的协同工作过程 |
| **知识溯源** | 点击引用标注，直接跳转查看原文出处 |

### 1.2 适用人群

| 用户类型 | 获得的内容风格 |
|---------|--------------|
| **本科生** | 通俗易懂的入门解读，带有类比和生活化比喻 |
| **硕士生** | 标准学术表达，包含方法对比和代码示例 |
| **博士生** | 前沿分析、研究空白识别、创新建议 |
| **教师/研究者** | 知识体系梳理、教学案例、完整参考文献 |

> **提示**：以上差异由您的**用户画像**自动驱动，无需手动切换。详见[第13章 个性化机制说明](#13-个性化机制说明)。

### 1.3 系统要求

| 项目 | 最低要求 |
|------|---------|
| **浏览器** | Chrome 90+ / Firefox 88+ / Safari 14+ / Edge 90+ |
| **网络** | 需要稳定的互联网连接（使用云端模型时） |
| **分辨率** | 推荐 1280×720 及以上 |

## 3 用户注册与登录

1. 在浏览器中打开系统首页
2. 点击右上角「**注册**」按钮
3. 填写注册表单：

| 字段 | 说明 |

点击页面右上角的用户头像 → 选择「**退出登录**」即可安全退出。

## 4 设置用户画像

> **重要**：用户画像是系统的核心特色，它决定了您获得的所有分析内容的深度和风格。首次登录后会引导您设置画像。

### 4.1 画像四个维度

#### 维度一：学历层次

| 选项 | 适合人群 | 内容策略 |
|------|---------|---------|
| **本科生** | 计算机相关专业本科在读 | 通俗解释 + 生活类比 + 入门建议 |
| **硕士生** | 硕士研究生在读 | 方法对比 + 实验分析 + 代码示例 |
| **博士生** | 博士研究生在读 | 前沿分析 + 研究空白 + 创新建议 |
| **教师/研究者** | 高校教师或科研人员 | 知识体系 + 教学案例 + 参考文献 |

#### 维度二：研究方向

选择您主要的研究领域，如 NLP（自然语言处理）、CV（计算机视觉）、RL（强化学习）、多模态等。这将影响检索排序权重和推荐策略。

#### 维度三：知识水平

| 选项 | 适合人群 | 术语密度 |
|------|---------|---------|
| **初级** | 对该领域了解较少 | < 5%（术语全解释） |
| **中级** | 有一定基础 | ~20%（关键术语解释） |
| **高级** | 深入研究 | ~40%（行业术语保留） |
| **专家** | 领域专家 | > 50%（原始术语） |

#### 维度四：偏好风格

| 选项 | 表达方式 | 适用场景 |
|------|---------|---------|
| **通俗** | 日常用语 + 比喻类比 | 快速入门、科普了解 |
| **均衡** | 标准学术表达 | 文献调研、课程报告 |
| **专业** | 正式学术 + 完整引用 | 论文写作、发表级综述 |

### 4.2 修改画像

1. 点击右上角用户头像 → 进入「**用户中心**」
2. 找到「**用户画像**」板块
3. 修改对应维度后点击「**保存**」
4. 修改后的画像立即生效，后续分析将使用新的设置

> **提示**：历史分析结果不受画像修改影响，仅对新分析生效。

## 5 智能论文检索

### 5.1 输入研究主题

1. 在首页中央的搜索框中输入研究主题（支持中英文）
   - 示例：`"Multi-Agent协同决策"`
   - 示例：`"large language model reasoning"`
2. 按回车键或点击「**搜索**」按钮
3. 系统会利用三路混合检索引擎寻找最相关的论文

### 5.2 理解检索结果

检索结果以论文卡片列表形式展示，每张卡片包含：

| 信息项 | 说明 |
|--------|------|
| **标题** | 完整论文标题 |
| **作者** | 前三位作者（点击展开可查看完整列表） |
| **摘要** | 截取前200字展示 |
| **关键词** | 最多展示5个关键词标签 |
| **相关度评分** | 与您搜索主题的匹配程度（百分比） |
| **推荐理由** | AI给出的一句话说���推荐原因 |

### 5.3 筛选与排序

检索结果页左侧有**筛选面板**，支持：

| 筛选项 | 说明 |
|--------|------|
| **年份范围** | 拖动滑块选择发表年份区间（2015-2026） |
| **引用数范围** | 设置最低和最高引用数 |
| **发表会议/期刊** | 多选下拉框，按顶级会议筛选 |

排序方式支持：
- **相关度排序**（默认）：最匹配的排在前面
- **发表时间排序**：最新论文排在前面
- **引用数排序**：高引用论文排在前面

### 5.4 分页浏览

搜索结果每页显示**10篇**论文，底部有页码导航。点击页码或「上一页/下一页」切换。

## 6 论文深度分析

### 6.1 进入论文详情

在检索结果页中，**点击任意论文卡片**即可进入论文详情页，查看完整信息：

- 完整标题和作者列表
- 完整摘要
- 发表年份、会议/期刊
- 引用数
- PDF链接（可跳转到原文）

### 6.2 启动AI分析

1. 在论文详情页点击「**智能分析**」按钮
2. 系统弹出分析确认弹窗，点击「**开始分析**」
3. 等待AI处理（通常**< 30秒**），期间可看到加载动画
4. 分析完成后展示**五维度分析卡片**

### 6.3 理解五维度分析结果

| 维度 | 内容说明 | 示例问题 |
|------|---------|---------|
| **研究问题** | 论文试图解决什么核心问题 | "该论文研究了Multi-Agent系统中如何有效协调多个Agent..." |
| **核心方法** | 论文提出的核心技术或架构 | "提出了基于LangGraph的图结构Agent编排框架..." |
| **主要实验** | 实验设置、数据集和关键结果 | "在5个基准数据集上进行了实验，对比了3种基线方法..." |
| **核心结论** | 论文的主要贡献和发现 | "证明了图结构编排相比线性编排，任务完成率提升23%..." |
| **局限性** | 论文承认的不足和未来方向 | "当前方法在实时性方面存在不足，未来将探索异步协作..." |

### 6.4 通俗解释（初级用户专属）

如果您的知识水平设为「**初级**」或「**中级**」，分析结果底部会额外显示「**通俗解释**」板块：

- 用日常语言重新阐述论文核心思想
- 使用生活化的类比帮助理解
- 避免复杂术语和公式

> **高级/专家用户**不会看到通俗解释板块，因为您不需要。

## 7 多论文对比分析

### 7.1 选择对比论文

1. 在检索结果页，勾选您想对比的论文（支持**2-5篇**）
2. 页面顶部会出现「**对比分析**」按钮
3. 点击按钮进入对比分析页面

### 7.2 查看对比结果

对比结果以**矩阵表格**形式展示，包含以下对比维度：

| 对比维度 | 内容 |
|---------|------|
| **方法对比** | 各论文使用的核心技术/架构对比 |
| **数据集对比** | 实验所用的数据集和评估基准对比 |
| **性能对比** | 关键性能指标的一对一比较 |
| **结论对比** | 核心结论和贡献的异同分析 |

对比结果下方还有**差异总结**文字描述，帮助快速理解各论文之间的异同。

### 7.3 矛盾发现

当不同论文之间存在**观点冲突**时，系统会自动标注：

- 矛盾点以**高亮橙色**显示
- 每条矛盾标注了**两篇论文各自的原始观点**
- 附带有**可能的原因解释**（如实验设置不同、数据集差异等）

## 8 个性化综述生成

> 这是系统的**核心功能**，6个AI智能体将协同工作，为您生成一份完整的文献综述。

### 8.1 启动综述生成

有两种方式启动：

**方式一：从检索结果页**
1. 在检索结果页选择论文范围
2. 点击「**生成综述**」按钮

**方式二：从对比分析页**
1. 完成对比分析后
2. 点击页面底部的「**基于对比结果生成综述**」按钮

### 8.2 观看Agent协同过程

点击生成后，页面会跳转到综述生成页，您将看到：

1. **Agent流程图**：显示6个AI智能体的实时工作状态
2. **状态面板**：各Agent的执行进度（等待 → 执行中 → 已完成）
3. **中间结果**：实时展示每个Agent产出的摘要信息

Agent工作顺序：

```
1. 协调者 Agent → 分解您的需求为子任务
2. 检索 Agent   → 找到最相关的论文
3. 分析 Agent   → 逐篇提取核心信息
4. 对比 Agent   → 对比不同论文的方法和结论（论文≥2篇时执行）
5. 生成 Agent   → 撰写个性化综述
6. 审核 Agent   → 核查引用和事实准确性
```

### 8.3 综述报告结构

生成的综述报告包含以下标准结构：

| 章节 | 内容 |
|------|------|
| **引言** | 研究背景和综述目的 |
| **研究现状** | 该领域的当前研究概况 |
| **方法对比** | 不同方法/架构的横向比较 |
| **研究趋势** | 未来发展方向和开放问题 |
| **参考文献** | 带引用标注的完整文献列表 |

### 8.4 个性化效果

同一研究主题，不同画像用户获得的综述报告会有明显不同：

| 用户画像 | 综述风格 |
|---------|---------|
| 本科生 + 初级 + 通俗 | 通俗易懂，大量类比，推荐入门资源 |
| 博士生 + 高级 + 专业 | 专业深度，前沿分析，研究空白，完整引用 |

> **演示场景**：可以请两位画像不同的用户分别生成同一主题的综述，对比差异展示系统个性化能力。

## 9 Agent协同可视化

### 9.1 什么是Agent可视化

在进行论文分析或综述生成时，系统会实时展示**6个AI智能体**的协同工作过程。您可以看到每个Agent：
- 正在做什么（等待/执行中/完成/失败）
- 产出了什么中间结果
- 花费了多少时间

### 9.2 流程图解读

Agent流程图使用**ECharts**技术渲染，节点颜色含义：

| 颜色 | 状态 | 说明 |
|------|------|------|
| **灰色** | 等待中 | 该Agent尚未开始执行 |
| **蓝色（闪烁）** | 执行中 | 该Agent正在工作 |
| **绿色** | 已完成 | 该Agent成功完成任务 |
| **红色** | 失败 | 该Agent执行失败（系统会自动降级处理） |

### 9.3 中间结果面板

当Agent执行时，右侧面板会实时展示中间结果摘要：

```
检索 Agent：       "找到 15 篇相关论文，筛选 Top10"
分析 Agent：       "已分析 8 / 10 篇"
对比 Agent：       "发现 2 处观点矛盾"
生成 Agent：       "正在生成综述..."
审核 Agent：       "审核通过，引用正确率 95%"
```

### 9.4 耗时统计

可视化页面底部有时耗统计柱状图，显示每个Agent的：
- 执行时长（毫秒）
- 开始/完成时间
- 总进度百分比

## 10 引用溯源

### 10.1 什么是引用溯源

综述报告中的每条引用都标注了来源。点击引用标记，即可溯源到原文信息。

### 10.2 使用方法

1. 在综述报告中，看到类似 `[1]`、`[2]` 的引用标注
2. **点击引用标注**，弹出引用溯源弹窗
3. 弹窗显示：
   - 引用来源的论文标题
   - 原文摘要片段
   - 「查看论文详情」链接（可跳转完整论文页）

### 10.3 知识溯源链路

完整的信息溯源路径：

```
综述引用标记 [1] → 点击跳转 → 原文片段弹窗 → 查看论文详情 → 论文原文 PDF
```

> 所有AI生成内容均可追溯到原始文献，确保信息的**可验证性**。

## 11 报告导出

### 11.1 支持格式

| 格式 | 说明 | 适用场景 |

导出的报告包含：
- 完整的综述正文（保留各级标题结构）
- 引用标注（保留跳转功能，PDF中为超链接）
- 生成时间和用户画像信息
- 「AI生成，仅供参考」提示标注

## 13 个性化机制说明

### 13.1 如何实现个性化

系统的个性化由**用户画像**驱动，通过以下三个层面生效：

```
用户画像
    │
    ├── 检索排序调整 ──→ 研究方向匹配的论文排名更高
    │
    ├── Prompt个性化注入 ──→ Agent的工作指令中加入您的画像信息
    │    例如：【重要】用户是硕士生，知识水平中级，请使用标准学术语言，需解释关键术语
    │
```

## 15 常见问题（FAQ）

### 15.1 注册与登录

**Q：忘记密码怎么办？**
A：目前版本请通过注册邮箱联系管理员重置密码。后续版本将支持自助找回密码。

**Q：登录状态能保持多久？**
A：登录有效期为**24小时**。超时后需重新登录。

**Q：可以在多台设备同时登录吗？**
A：可以。同一账号可在多台设备上同时使用。

### 15.2 检索与分析

**Q：搜索不到我想要的论文怎么办？**
A：尝试以下方法：
1. 使用更具体的关键词（如加上"survey"或"2024"）
2. 尝试英文关键词（系统同时支持中英文检索）
3. 调整筛选条件（扩大年份范围）

**Q：分析结果不准确怎么办？**
A：AI分析基于论文摘要和元数据进行推理，对于复杂论文可能有偏差。您可以通过修改用户画像来调整分析深度。未来版本将支持PDF全文分析。

**Q：单篇论文分析需要多长时间？**
A：通常**< 30秒**。综述生成需要**< 60秒**（含6个Agent协同全过程）。

**Q：为什么分析结果没有通俗解释？**
A：通俗解释仅对知识水平为「初级」和「中级」的用户显示。高级/专家用户不会看到此板块。

### 15.3 Agent可视化

**Q：Agent流程图中某个节点变红了怎么办？**
A：红色代表该Agent执行失败，但系统会自动降级处理，不影响整体结果。您仍可获得部分分析结果。

**Q：SSE连接断了怎么办？**
A：系统会自动重连（3秒间隔，最多重试5次）。如果持续断开，请检查网络连接。

### 15.4 综述生成

**Q：生成的综述可以修改吗？**
A：当前版本支持导出Word后进行手动修改。系统内编辑器功能计划中（P2优先级）。

**Q：综述引用的文献准确吗？**
A：系统有审核Agent进行引用核查，引用正确率目标 > 90%。您也可通过引用溯源功能手动验证。

**Q：为什么不同用户同一主题的综述不一样？**
A：这正是系统的个性化特色！综述内容会根据用户画像（学历、知识水平、偏好风格）动态调整。见[第13章 个性化机制说明](#13-个性化机制说明)。

### 15.5 部署与运维

**Q：Docker部署失败怎么办？**
A：请检查：
1. Docker和Docker Compose版本是否满足要求
2. 端口是否被占用（80/3306/6379/8000/8080）
3. 环境变量 .env 文件是否正确配置
4. 运行 `docker compose logs` 查看具体错误日志

**Q：可以不使用Docker部署吗？**
A：可以，但需要手动配置MySQL、Redis、Java、Python环境。推荐使用Docker Compose一键部署。

**Q：论文数据如何更新？**
A：通过管理后台的论文导入功能或直接操作MySQL数据库和ChromaDB向量库。

**Q：服务器最低配置要求？**
A：使用软件方模型时，最低4GB内存 + 10GB磁盘即可运行。
