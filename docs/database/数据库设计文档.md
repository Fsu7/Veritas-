## 1 文档概述

### 1.1 编写目的

本文档详细定义科研文献智能助手系统中全部数据库的设计规范，包括MySQL关系数据库、Redis缓存数据库、ChromaDB向量数据库的表结构、字段定义、索引策略、缓存Schema、向量Schema、数据一致性策略等内容，为系统开发提供完整的数据层设计蓝图。

### 1.2 适用范围

本文档覆盖系统全部4个数据库的设计：

| 数据库 | 类型 | 用途 | 访问方 |
|--------|------|------|--------|
| MySQL 8.0 | 关系数据库 | 论文元数据、用户数据、会话数据、分析结果 | Java后端（Spring Data JPA） |
| Redis 7.0 | 缓存数据库 | 热点数据缓存、会话状态、Agent状态、Token黑名单 | Java后端（Spring Data Redis） |
| ChromaDB 0.5+ | 向量数据库 | 论文语义向量存储与检索 | Python AI服务（chromadb客户端） |
| Neo4j 5.x | 图数据库 | 论文实体关系、方法改进链、知识图谱推理 | Python AI服务（neo4j驱动） |

### 1.3 设计原则

| 原则 | 说明 | 体现 |
|------|------|------|
| **数据分层存储** | 关系数据、缓存数据、向量数据分离存储 | MySQL存关系、Redis存缓存、Chroma存向量 |
| **缓存优先** | 读操作先查缓存，减轻数据库压力 | Cache-Aside Pattern |
| **数据一致性** | 写操作先更新DB，再失效缓存 | 写后删策略 |
| **适度冗余** | 允许少量冗余换取查询性能 | JSON字段存储灵活数据 |
| **可扩展性** | 预留扩展字段，支持未来需求变更 | profile_data JSON字段 |
| **安全优先** | 敏感数据加密存储，数据隔离访问 | BCrypt密码、数据隔离 |

### 2.2 数据流转关系

```
写入流程：
                                   
  用户操作 → Java后端 → MySQL（持久化）
                  │
                  └──→ Redis缓存失效（写后删策略）
                  
  论文导入 → Java后端 → MySQL（论文元数据）
                  │
                  └──→ Python AI服务 → ChromaDB（向量存储）
                  
  分析请求 → Java后端 → MySQL（创建AnalysisResult）
                  │
                  ├──→ Python AI服务 → ChromaDB（向量检索）
                  │         │
                  │         └──→ Redis（Agent状态更新）
                  │
                  └──→ MySQL（保存分析结果）
                       └──→ Redis（缓存分析结果）

读取流程：

  用户查询 → Java后端 → Redis缓存（命中？返回）
                  │
                  └──→ MySQL（未命中时查询）
                       └──→ Redis（回填缓存）
                       
  论文检索 → Java后端 → Redis缓存（命中？返回）
                  │
                  └──→ Python AI服务 → ChromaDB（向量检索）
                       └──→ Java后端 → Redis（缓存检索结果）
```

### 3.3 users表 — 用户表

#### 3.3.1 表说明

| 项目 | 说明 |
|------|------|
| **用途** | 存储用户基础账号信息 |
| **关联** | 1:1 → user_profiles，1:N → sessions，1:N → paper_favorites |

#### 3.3.2 字段定义

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|

| `user_id` | VARCHAR(100) | UNIQUE, NOT NULL | - | 用户唯一ID（UUID格式，如 `usr_20240523_001`） |
| `username` | VARCHAR(100) | NOT NULL | - | 用户名 |
| `email` | VARCHAR(200) | NULL | NULL | 邮箱地址 |
| `password_hash` | VARCHAR(200) | NOT NULL | - | BCrypt加密后的密码哈希 |
| `created_at` | TIMESTAMP | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |

#### 3.3.3 索引设计

| 索引名 | 类型 | 字段 | 说明 |
|--------|------|------|------|
| `PRIMARY` | 主键 | `id` | 自增主键 |
| `uk_user_id` | 唯一索引 | `user_id` | 用户ID唯一，登录查询使用 |

#### 3.3.4 JPA Entity定义

```java
@Entity
@Table(name = "users")
@Data
@NoArgsConstructor
@AllArgsConstructor
@Builder
public class User {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "user_id", unique = true, nullable = false, length = 100)
    private String userId;

    @Column(nullable = false, length = 100)
    private String username;

    @Column(length = 200)
    private String email;

    @Column(name = "password_hash", nullable = false, length = 200)
    private String passwordHash;

    @Column(name = "created_at", updatable = false)
    private LocalDateTime createdAt;

    @PrePersist
    protected void onCreate() {
        createdAt = LocalDateTime.now();
    }
}
```

#### 3.5.5 全文检索查询方式

```sql
-- 自然语言模式（默认）
SELECT * FROM papers 
WHERE MATCH(title, abstract) AGAINST('Multi-Agent' IN NATURAL LANGUAGE MODE);

-- 布尔模式（支持高级搜索语法）
SELECT * FROM papers 
WHERE MATCH(title, abstract) AGAINST('+Multi-Agent +survey' IN BOOLEAN MODE);

-- 带条件过滤的全文检索
SELECT * FROM papers 
WHERE MATCH(title, abstract) AGAINST(:keyword IN NATURAL LANGUAGE MODE)
  AND (:yearFrom IS NULL OR year >= :yearFrom)
  AND (:yearTo IS NULL OR year <= :yearTo)
  AND (:venue IS NULL OR venue = :venue)
ORDER BY MATCH(title, abstract) AGAINST(:keyword IN NATURAL LANGUAGE MODE) DESC;
```

## 4 Redis缓存数据库设计

### 4.1 Redis基本信息

| 项目 | 值 |
|------|-----|
| **版本** | Redis 7.0+ |
| **部署** | Docker容器（redis:7-alpine镜像） |
| **端口** | 6379 |
| **持久化** | AOF持久化（appendonly yes），确保JWT黑名单等关键数据不丢失 |
| **Java访问层** | Spring Data Redis |
| **序列化** | Key: StringRedisSerializer, Value: GenericJackson2JsonRedisSerializer |

### 4.2 缓存Key命名规范

```
格式：{业务域}:{数据类型}:{唯一标识}

示例：
  user:profile:usr_20240523_001       # 用户画像
  user:info:usr_20240523_001           # 用户基础信息
  paper:detail:arxiv_2024_001          # 论文详情
  search:result:abc123def456           # 检索结果
  analysis:result:anl_20240523_001     # 分析结果
  session:state:ses_20240523_001       # 会话状态
  agent:state:anl_20240523_001         # Agent执行状态
  auth:blacklist:eyJhbG...            # JWT Token黑名单
```

### 4.3 缓存空间详情

| 缓存空间 | Key格式 | TTL | 数据结构 | 失效策略 | 命中率预期 | 存储大小预估 |
|---------|---------|-----|---------|---------|-----------|------------|
| **用户画像** | `user:profile:{userId}` | 1小时 | String(JSON) | 画像更新时主动失效（@CacheEvict） | >50% | ~2KB/条 |
| **用户信息** | `user:info:{userId}` | 1小时 | String(JSON) | 用户信息更新时主动失效 | >60% | ~0.5KB/条 |
| **论文详情** | `paper:detail:{paperId}` | 30分钟 | String(JSON) | 论文导入时批量失效 | >40% | ~5KB/条 |
| **论文搜索** | `search:result:{queryHash}` | 10分钟 | String(JSON) | 新论文导入时清空搜索缓存 | >30% | ~50KB/条 |
| **分析结果** | `analysis:result:{analysisId}` | 30分钟 | String(JSON) | 结果更新时失效 | >20% | ~100KB/条 |
| **会话状态** | `session:state:{sessionId}` | 2小时 | String(JSON) | 会话完成/删除时失效 | >80% | ~1KB/条 |
| **Agent状态** | `agent:state:{analysisId}` | 5分钟 | Hash | 分析完成后自然过期 | 实时数据 | ~2KB/条 |
| **Token黑名单** | `auth:blacklist:{tokenHash}` | Token剩余有效期 | String | 自然过期 | - | ~0.1KB/条 |

### 4.4 缓存数据结构详解

#### 4.4.1 用户画像缓存

```redis
# Key格式
user:profile:{userId}

# 值（JSON序列化）
SET user:profile:usr_001 '{"educationLevel":"master","researchField":"NLP","knowledgeLevel":"intermediate","preferredStyle":"balanced","profileData":{"researchStage":"survey","mathLevel":"medium"}}'
```

```redis
# 值（JSON序列化）
```

```redis
# 值（JSON序列化，包含完整分析结果）
```

```redis

# 值
```

### 4.5 RedisConfig配置

```java
@Configuration
@EnableCaching
public class RedisConfig {

    @Bean
    public RedisCacheManager cacheManager(RedisConnectionFactory factory) {
        // 默认TTL：30分钟
        RedisCacheConfiguration defaultConfig = RedisCacheConfiguration.defaultCacheConfig()
            .entryTtl(Duration.ofMinutes(30))
            .serializeKeysWith(RedisSerializationContext.SerializationPair
                .fromSerializer(new StringRedisSerializer()))
            .serializeValuesWith(RedisSerializationContext.SerializationPair
                .fromSerializer(new GenericJackson2JsonRedisSerializer()));

        // 各缓存空间的自定义TTL
        Map<String, RedisCacheConfiguration> cacheConfigurations = new HashMap<>();
        cacheConfigurations.put("userProfile", defaultConfig.entryTtl(Duration.ofHours(1)));
        cacheConfigurations.put("userInfo", defaultConfig.entryTtl(Duration.ofHours(1)));
        cacheConfigurations.put("paperDetail", defaultConfig.entryTtl(Duration.ofMinutes(30)));
        cacheConfigurations.put("paperSearch", defaultConfig.entryTtl(Duration.ofMinutes(10)));
        cacheConfigurations.put("analysisResult", defaultConfig.entryTtl(Duration.ofMinutes(30)));
        cacheConfigurations.put("sessionState", defaultConfig.entryTtl(Duration.ofHours(2)));

        return RedisCacheManager.builder(factory)
            .cacheDefaults(defaultConfig)
            .withInitialCacheConfigurations(cacheConfigurations)
            .build();
    }
}
```

### 5.7 VectorStoreService核心代码

```python
class VectorStoreService:
    """Chroma向量数据库服务"""

    async def initialize(self):
        self.client = chromadb.PersistentClient(
            path=self.settings.CHROMA_PATH or "./data/vector_db"
        )
        self.collection = self.client.get_or_create_collection(
            name="papers",
            metadata={
                "hnsw:space": "cosine",
                "hnsw:M": 16,
                "hnsw:construction_ef": 200
            }
        )

    async def search(self, embedding: list, top_k: int = 10,
                     filters: dict = None) -> list:
        # 构建过滤条件
        where_filter = None
        if filters:
            conditions = []
            if filters.get("yearFrom"):
                conditions.append({"year": {"$gte": filters["yearFrom"]}})
            if filters.get("yearTo"):
                conditions.append({"year": {"$lte": filters["yearTo"]}})
            if filters.get("venue"):
                conditions.append({"venue": {"$eq": filters["venue"]}})
            if conditions:
                where_filter = {"$and": conditions} if len(conditions) > 1 else conditions[0]

        results = self.collection.query(
            query_embeddings=[embedding],
            n_results=top_k,
            where=where_filter,
            include=["metadatas", "distances", "documents"]
        )

        # 格式化结果
        formatted = []
        for i in range(len(results["ids"][0])):
            formatted.append({
                "paperId": results["metadatas"][0][i].get("paper_id"),
                "title": results["metadatas"][0][i].get("title"),
                "abstract": results["documents"][0][i],
                "score": 1 - results["distances"][0][i],  # cosine距离→相似度
                "year": results["metadatas"][0][i].get("year"),
                "venue": results["metadatas"][0][i].get("venue")
            })
        return formatted
```

```cypher
CREATE CONSTRAINT paper_id IF NOT EXISTS
FOR (p:Paper) REQUIRE p.paper_id IS UNIQUE;

CREATE CONSTRAINT method_name IF NOT EXISTS
FOR (m:Method) REQUIRE m.name IS UNIQUE;

CREATE CONSTRAINT concept_name IF NOT EXISTS
FOR (c:Concept) REQUIRE c.name IS UNIQUE;

// 创建论文节点
CREATE (p:Paper {
    paper_id: 'arxiv_2024_001',
    title: 'Multi-Agent Systems: A Survey',
    year: 2024,
    venue: 'AAAI'
})

// 创建方法节点
CREATE (m:Method {name: 'LangGraph', category: 'Agent Framework'})

// 创建关系：论文使用方法
MATCH (p:Paper {paper_id: 'arxiv_2024_001'})
MATCH (m:Method {name: 'LangGraph'})
CREATE (p)-[:USES {context: 'Agent编排框架'}]->(m)

// 图谱检索：查找某方法的所有改进链
MATCH (m:Method {name: 'LangGraph'})<-[:IMPROVES*1..3]-(improved)
RETURN improved.name, improved.category

// 图谱检索：查找相关论文的方法关系网络
MATCH (p:Paper)-[:USES]->(m:Method)-[:BELONGS_TO]->(c:Concept)
WHERE c.name = 'Multi-Agent'
RETURN p.title, m.name, c.name
```

## 7 数据一致性保障

### 6.1 Cache-Aside Pattern（旁路缓存）

```
读操作：
1. 先读Redis缓存
2. 缓存命中 → 直接返回
3. 缓存未命中 → 查MySQL → 回填Redis → 返回

写操作：
1. 先更新MySQL
2. 再删除Redis缓存（@CacheEvict）
3. 下次读取时从MySQL重新加载
```

### 6.2 各数据场景一致性策略

| 数据场景 | 一致性策略 | 说明 |
|---------|-----------|------|
| **用户画像更新** | 双重失效 | userProfile + userProfileJson同步失效 |
| **论文导入** | 搜索缓存清空 | 新论文影响搜索结果，清空search:*缓存 |
| **分析结果更新** | 写后删 | 先更新MySQL，再删除Redis缓存 |
| **Agent状态** | 仅Redis存储 | 不经过MySQL，实时性要求高，自然过期 |
| **会话状态变更** | 写后删 | 先更新MySQL，再删除Redis缓存 |
| **论文收藏/取消** | 无缓存 | 直接操作MySQL，不涉及缓存 |

### 6.3 跨数据库数据同步

```
论文数据同步流程：

arXiv API下载
    │
    ▼
数据清洗（去重、格式统一）
    │
    ├──→ MySQL: INSERT INTO papers (元数据)
    │
    └──→ Python AI服务:
         ├── EmbeddingService.encode() → 768维向量
         ├── VectorStoreService.add_papers() → ChromaDB
         └── MySQL paper_id = ChromaDB metadata.paper_id

删除论文流程：

    ├──→ MySQL: DELETE FROM papers WHERE paper_id = ?
    └──→ ChromaDB: collection.delete(ids=[paper_id])
```

```sql
    user_id VARCHAR(100) NOT NULL COMMENT '关联用户ID',
    paper_id VARCHAR(100) NOT NULL COMMENT '关联论文ID',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '收藏时间',
    UNIQUE KEY uk_user_paper (user_id, paper_id),
    FOREIGN KEY (user_id) REFERENCES users(user_id) ON DELETE CASCADE,
    FOREIGN KEY (paper_id) REFERENCES papers(paper_id) ON DELETE CASCADE,
    INDEX idx_user_id (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='论文收藏表';
```
