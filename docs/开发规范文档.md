## 1 引言

### 1.1 编写目的

本开发规范文档为 XH-202630 科研文献智能助手项目制定统一的开发标准，旨在：

1. 保证代码质量与一致性，降低协作成本
2. 规范跨系统（Java后端、Python AI服务、前端）的开发标准
3. 为AI Coding辅助开发提供标准化上下文
4. 减少因规范不统一导致的Bug和返工

### 1.2 适用范围

本文档适用于项目所有开发活动，覆盖：

- 前端（Vue3 + TypeScript）
- Java后端（Spring Boot 3.2+）
- Python AI服务（FastAPI + LangGraph）
- 数据库（MySQL、Redis、ChromaDB、Neo4j）
- 部署（Docker + Docker Compose）

### 1.3 规范原则

| 原则 | 说明 |
|------|------|
| **一致性优先** | 同一技术栈内风格统一，跨技术栈接口约定一致 |
| **实用优先** | 规范服务于开发效率，不增加不必要的形式负担 |
| **AI友好** | 代码结构和命名对AI Coding工具友好，便于生成和理解 |
| **渐进严格** | P0功能严格执行规范，P1/P2功能允许适度灵活 |

### 1.4 强制等级标识

| 标识 | 含义 | 违反后果 |
|------|------|---------|
| **【强制】** | 必须遵守，不得违反 | Code Review打回 |
| **【推荐】** | 建议遵守，特殊情况可偏离 | 需在代码中注释说明原因 |
| **【参考】** | 仅供参考，自由选择 | 无 |

## 3 代码规范

### 3.1 通用编码规范

```
【强制】通用规范：

1. 字符编码：所有源文件使用 UTF-8 编码
2. 换行符：使用 LF（Unix风格），不使用 CRLF
3. 缩进：
   - Java: 4空格
   - Python: 4空格
   - TypeScript/Vue: 2空格
4. 行宽：不超过120字符
5. 尾行：文件末尾保留一个空行
6. 空行：类/方法之间保留一个空行，逻辑分组之间保留一个空行
7. 空格：运算符两侧、逗号后、关键字后必须有空格
8. 大括号：Java/TypeScript使用同行大括号风格，Python不使用大括号
```

## 4 API设计规范

### 4.1 RESTful API规范

```
【强制】API设计原则：

1. URL使用名词复数形式：/api/papers, /api/sessions
2. HTTP方法语义：
   ├── GET    - 查询资源，不修改状态
   ├── POST   - 创建资源或触发操作
   ├── PUT    - 全量更新资源
   ├── DELETE - 删除资源
   └── 禁止使用GET修改数据
3. URL层级不超过3层：/api/{resource}/{id}/{sub-resource}
4. 路径参数使用camelCase（与Java保持一致）：/api/users/{userId}
5. 查询参数使用camelCase：?pageSize=10&sortBy=relevance
```

### 4.2 统一响应格式

```
【强制】所有API必须使用统一的响应格式：

成功响应：
{
  "code": 200,
  "message": "success",
  "data": { ... },
  "timestamp": "2026-05-23T10:00:00Z"
}

错误响应：

  "code": 401,
  "message": "认证失败：Token无效或已过期",
  "error": "AUTH_TOKEN_INVALID",
  "timestamp": "2026-05-23T10:00:00Z"
}

分页响应：
{
  "code": 200,
  "data": {
    "items": [ ... ],
    "pagination": {
      "page": 1,
      "size": 10,
      "total": 85,
      "totalPages": 9
    }
  }
}

SSE事件格式：
event: agent_state_update
data: {"agent_name": "retriever", "status": "running", "progress": 0.6}
```

### 4.3 HTTP状态码规范

```
【强制】正确使用HTTP状态码：

  状态码 │ 含义                  │ 使用场景
 ────────┼──────────────────────┼──────────────────────────
  200    │ 成功                  │ GET查询、PUT更新成功
  201    │ 创建成功              │ POST创建资源成功
  204    │ 无内容                │ DELETE删除成功
  400    │ 请求参数错误          │ 参数校验失败
  401    │ 未认证                │ Token无效或缺失
  403    │ 无权限                │ 访问他人数据
  404    │ 资源不存在            │ 查询的ID不存在
  409    │ 冲突                  │ 用户名重复
  422    │ 不可处理实体          │ 业务逻辑校验失败
  500    │ 服务器内部错误        │ 未捕获的异常
  502    │ 网关错误              │ AI服务不可用
  503    │ 服务不可用            │ 服务正在启动
```

### 7.5 枚举与ORM映射规范

```
【强制】Java枚举设计原则：

Java层保持 `UPPER_SNAKE_CASE`（符合Java命名规范），数据库存储 `lowercase` 字符串（符合MySQL规范），
通过 `AttributeConverter`（`@Converter(autoApply=true)`）实现双向自动转换。

┌────────────────────────────────────────────────────────────────┐
│              枚举ORM映射架构                                   │
│                                                                │
│  Java枚举 (UPPER_SNAKE_CASE)          数据库 (lowercase)       │
│                                                                │
│  AnalysisStatus.PENDING  ── convertToDatabaseColumn() ──→     │
│                                    String dbValue = "pending"  │
│                                                                │
│  AnalysisStatus.PENDING  ←── convertToEntityAttribute() ──     │
│                                    String dbData = "pending"   │
└────────────────────────────────────────────────────────────────┘

【强制】枚举定义规范：

1. 所有枚举实现 DbValueEnum 接口（定义 getDbValue() 返回数据库值）
2. 枚举值使用 UPPER_SNAKE_CASE（PENDING, ACTIVE, MASTER）
3. dbValue 使用 lowercase（"pending", "active", "master"）
4. 包含 dbValue 和 label（中文描述）两个私有字段

示例：
  public enum AnalysisStatus implements DbValueEnum {
      PENDING("pending"),
      PROCESSING("processing"),
      COMPLETED("completed"),
      FAILED("failed");
  
      @Getter private final String dbValue;
      // getDbValue() from DbValueEnum interface
  }

【强制】Converter规范：

1. 每个枚举创建对应的 Converter 类，继承 AbstractEnumConverter
2. 使用 @Converter(autoApply=true) 注解（自动生效，Entity字段无需@Convert）
3. Converter命名：{EnumName}Converter
4. 存放在 enums/ 目录下，与枚举类同包

示例：
  @Converter(autoApply = true)
  public class AnalysisStatusConverter extends AbstractEnumConverter<AnalysisStatus> {
      public AnalysisStatusConverter() { super(AnalysisStatus.class); }
  }

【强制】Entity中使用规范：

Entity中枚举字段只需声明字段类型，无需 @Enumerated 或 @Convert 注解：

  // 正确 ✅  — Converter自动处理
  @Column(nullable = false, length = 20)
  private AnalysisStatus status;

  // 错误 ❌  — 不再需要 @Enumerated
  // @Enumerated(EnumType.STRING)
  // @Column(nullable = false, length = 20)
  // private AnalysisStatus status;

【强制】数据库列类型说明：

由于 AttributeConverter 将枚举映射为 String 存储，Hibernate（ddl-auto=update）会自动执行
ALTER TABLE ... MODIFY COLUMN ... VARCHAR(20)，将原有 ENUM 列转换为 VARCHAR。
这是正常行为，DDL脚本中保留ENUM定义作为参考。

【强制】跨系统字段命名转换：

Java (camelCase + UPPER_SNAKE_CASE枚举) → JSON (snake_case + lowercase枚举值)：
  - educationLevel: MASTER → "education_level": "master"
  - analysisStatus: PENDING → "analysis_status": "pending"
  
  通过 @JsonProperty 注解实现字段名转换，枚举值通过 getDbValue() 获取小写。
```

### 7.6 Spring Boot全局Jackson配置规范

```
【强制】所有API响应统一使用 snake_case 命名，禁止在每个DTO上重复使用 @JsonProperty：

┌────────────────────────────────────────────────────────────────────┐
│                 Java(camelCase) → JSON(snake_case)                 │
│                                                                    │
│  UserService.java              HTTP Response JSON                  │
│  ─────────────────            ──────────────────                  │
│  userId            ────────→   "user_id": "usr_001"                │

│  paperId           ────────→   "paper_id": "arxiv_001"             │
│  createdAt         ────────→   "created_at": "2026-06-02 10:00:00" │
└────────────────────────────────────────────────────────────────────┘

【强制】application.yml全局Jackson配置（必须包含以下5项）：

  spring:
    jackson:
      date-format: yyyy-MM-dd HH:mm:ss           # LocalDateTime序列化格式
      time-zone: Asia/Shanghai                   # 统一时区
      default-property-inclusion: non_null       # 忽略null字段，减小payload
      property-naming-strategy: SNAKE_CASE       # ★ 核心：全局snake_case
      mapper:
        accept-case-insensitive-enums: true      # ★ 枚举大小写不敏感

【推荐】Redis缓存ObjectMapper规范：

  Redis序列化需独立配置ObjectMapper，必须：
  ├── 1. 注册 JavaTimeModule 处理 LocalDateTime
  ├── 2. 禁用 WRITE_DATES_AS_TIMESTAMPS（避免序列化为时间戳）
  ├── 3. 启用 default typing（保留多态类型信息）
  └── 4. 实现 GenericJackson2JsonRedisSerializer Bean

  示例（RedisConfig.java）：

    @Bean
    public GenericJackson2JsonRedisSerializer jsonRedisSerializer() {
        ObjectMapper om = new ObjectMapper();
        om.registerModule(new JavaTimeModule());
        om.disable(SerializationFeature.WRITE_DATES_AS_TIMESTAMPS);
        om.activateDefaultTyping(
                LaissezFaireSubTypeValidator.instance,
                ObjectMapper.DefaultTyping.NON_FINAL,
                JsonTypeInfo.As.PROPERTY);
        return new GenericJackson2JsonRedisSerializer(om);
    }

  原因：Spring Boot默认的Jackson2JsonRedisSerializer不支持LocalDateTime，
  会序列化为时间戳或数组，导致反序列化失败。

【强制】Entity与DTO字段分离原则（结合7.2）：

  ├── Entity（entity/）: JPA实体，字段保持 camelCase Java命名
  ├── Request/Response DTO（dto/）: 字段保持 camelCase Java命名
  ├── 禁止在Entity上添加 @JsonProperty（避免污染ORM层）
  ├── 禁止在DTO上逐个添加 @JsonProperty（依赖全局配置）
  └── 特殊场景（如需兼容历史接口）才使用 @JsonProperty 覆盖

【强制】枚举反序列化大小写规范：

  ├── 数据库存储：lowercase（pending, active, master）
  ├── Java枚举：UPPER_SNAKE_CASE（PENDING, ACTIVE, MASTER）
  ├── JSON请求：接受 ACTIVE / active / Active 等任意大小写
  └── 通过 spring.jackson.mapper.accept-case-insensitive-enums: true 实现

【强制】Controller层使用MapStruct的Mapper实现Entity↔DTO转换（结合7.2）：

  @Mapper(componentModel = "spring", uses = {JsonStringListHelper.class})
  public interface PaperMapper {
      @Mapping(target = "authors", source = "authors", qualifiedByName = "jsonToList")
      PaperResponse toResponse(Paper paper);
  }

  MapStruct生成的实现类路径：target/generated-sources/annotations/
  若出现 FilerException: Source file already created，需执行 mvn clean 清理。

【推荐】MockMvc测试中需要显式配置ObjectMapper：

  @BeforeEach
  void setUp() {
      objectMapper = new ObjectMapper()
              .setPropertyNamingStrategy(PropertyNamingStrategies.SNAKE_CASE)
              .enable(MapperFeature.ACCEPT_CASE_INSENSITIVE_ENUMS)
              .registerModule(new JavaTimeModule())
              .disable(SerializationFeature.WRITE_DATES_AS_TIMESTAMPS);
      MappingJackson2HttpMessageConverter converter =
              new MappingJackson2HttpMessageConverter(objectMapper);
      mockMvc = MockMvcBuilders.standaloneSetup(controller)
              .setControllerAdvice(new GlobalExceptionHandler())
              .setMessageConverters(converter)
              .build();
  }
```

### 7.7 IDE团队共享配置规范（VS Code / TRAE / Eclipse JDT-LS）

```
【背景】TRAE 是基于 VS Code 的 AI IDE，其 Java 支持依赖 Eclipse JDT-LS
（Language Support for Java 扩展）。该扩展底层使用 Eclipse JDT 作为
Java 编译器，与 MapStruct 等注解处理器集成时存在已知问题：

  javax.annotation.processing.FilerException: Source file already created: PaperMapperImpl.java

【强制】根因：
  Eclipse JDT 默认按 maxCompiledUnitsAtOnce（默认256）切分编译单元，
  当多个 Mapper 文件分布在不同编译单元且引用同一 JsonStringListHelper
  时，第二个单元尝试创建同名 Impl 文件时 Filer 拒绝写入。

【强制】修复方案（项目级配置，提交到 Git）：

  1. 在 backend/.vscode/settings.json 添加（已落地）：

     {
       "java.jdt.ls.vmargs": "-DmaxCompiledUnitsAtOnce=10000 -XX:+UseG1GC -Xmx2G -Xms256m",
       "java.configuration.updateBuildConfiguration": "automatic",
       "java.jdt.ls.annotationProcessing.enabled": true,
       "java.jdt.ls.cleanOutputFolder": "clean"
     }

     关键参数说明：

     ├── -Xmx2G -Xms256m                   提升大项目的内存，避免 OOM
     └── cleanOutputFolder=clean           每次编译前清理 target/generated-sources

  2. 在 .gitignore 添加反向规则（已落地）：

     原因：禁用 Maven 增量编译，强制每次 clean compile，避免增量缓存
```

```
from string import Template

class PromptManager:
    def __init__(self, prompts_dir: str = "prompts"):
        self.prompts_dir = prompts_dir
        self._templates: Dict[str, str] = {}
    
    async def load_templates(self):
        """启动时加载所有Prompt模板"""
        for file in Path(self.prompts_dir).glob("*.txt"):
            self._templates[file.stem] = file.read_text(encoding="utf-8")
    
    def get_prompt(self, agent_name: str, **kwargs) -> str:
        """获取渲染后的Prompt"""
        template = self._templates.get(agent_name)
        if not template:
            raise ValueError(f"Prompt模板不存在: {agent_name}")
        return Template(template).safe_substitute(**kwargs)
```

### 9.3 LangGraph工作流规范

```

def build_workflow() -> StateGraph:
    workflow = StateGraph(AgentState)
    
    # 添加节点
    workflow.add_node("coordinate", coordinator.execute)
    workflow.add_node("retrieve", retriever.execute)
    workflow.add_node("analyze", analyzer.execute)
    workflow.add_node("compare", comparer.execute)
    workflow.add_node("generate", generator.execute)
    workflow.add_node("review", reviewer.execute)
    
    # 定义边
    workflow.set_entry_point("coordinate")
    workflow.add_edge("coordinate", "retrieve")
    workflow.add_edge("retrieve", "analyze")
    workflow.add_conditional_edges("analyze", should_compare, {
        True: "compare",
        False: "generate"
    })
    workflow.add_edge("compare", "generate")
    workflow.add_conditional_edges("review", should_regenerate, {
        True: "generate",
        False: END
    })
    workflow.add_edge("generate", "review")
```

```
4. 环境变量通过.env文件注入，不硬编码在docker-compose.yml
5. 端口映射：仅暴露必要端口（80, 8080, 8000）

启动顺序：
  mysql(redis) → healthy → ai-service → healthy → java-backend → frontend
```
