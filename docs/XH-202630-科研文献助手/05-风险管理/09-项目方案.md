# XH-202630 项目方案：科研文献智能助手

## 基于多智能体协同的领域知识个性化生成与决策系统

> **课题编号**：XH-202630
> **发榜单位**：上海云之脑智能科技有限公司（科大讯飞子公司）
> **应用领域**：科研文献智能分析
> **创建日期**：2026年5月23日
> **文档版本**：v1.0

---

## 一、项目定义

### 一句话描述

> 构建一个多智能体协同的科研文献分析系统，研究者输入研究主题后，多个AI Agent自动协作完成**文献检索、内容分析、知识提取、综述生成**，并根据研究者的方向和水平输出个性化的文献分析报告。

### 解决的核心问题

```
传统科研文献分析的痛点：

┌─────────────────────────────────────────────────────┐
│                                                     │
│  😫 "我想研究Multi-Agent，但论文太多了看不完"        │
│  😫 "每篇论文都要读几十页，太耗时了"                 │
│  😫 "不知道哪些论文是必读的，哪些可以跳过"           │
│  😫 "不同论文的观点怎么对比？有没有矛盾？"           │
│  😫 "写文献综述要花好几天，格式还容易出错"           │
│  😫 "我是本科生，很多论文看不懂"                     │
│  😫 "我是博士生，需要更深入的分析"                   │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 系统解决方案

```
用户输入研究主题
       ↓
┌──────────────────────────────────────────┐
│  多智能体协同工作（自动完成以下任务）      │
│                                          │
│  📚 检索Agent：从论文库中找到相关论文      │
│  🔍 分析Agent：提取每篇论文的核心内容      │
│  📊 对比Agent：对比不同论文的观点和方法    │
│  ✍️ 生成Agent：生成个性化文献综述          │
│  ✅ 审核Agent：检查内容的准确性和引用      │
│                                          │
└──────────────────────────────────────────┘
       ↓
输出：个性化文献分析报告
├── 核心论文推荐（附推荐理由）
├── 研究现状总结
├── 方法对比分析
├── 研究趋势与空白
└── 个性化学习路线
```

## 二、核心功能清单

### 功能1：智能文献检索与推荐

| 项目 | 内容 |
|------|------|
| **输入** | 用户输入研究主题（如"Multi-Agent协同决策"） |
| **处理** | 检索Agent从论文库中语义检索相关文献，按相关性排序 |
| **输出** | Top10相关论文列表，每篇附带：标题、摘要、关键词、相关性评分、推荐理由 |
| **个性化** | 根据用户的研究方向筛选最相关的子领域论文 |

**演示效果：**

| **个性化** | 初级用户看到通俗解释，高级用户看到技术细节 |

### Agent角色详细设计

| Agent | 角色比喻 | 输入 | 输出 | 核心工具 |
|-------|---------|------|------|---------|
| **协调者** | 项目经理 | 用户问题 + 用户画像 | 任务分配指令 + 最终回答 | 任务分解器 |
| **检索Agent** | 图书管理员 | 研究主题关键词 | 相关论文列表（Top10） | 向量检索、关键词搜索 |
| **分析Agent** | 论文审稿人 | 论文全文/摘要 | 结构化分析卡片 | 文本提取、信息抽取 |
| **对比Agent** | 对比研究员 | 2-5篇论文分析 | 对比表格 + 差异总结 | 对比引擎 |
| **生成Agent** | 学术写手 | 分析结果 + 用户画像 | 个性化综述报告 | 大模型生成 |
| **审核Agent** | 学术编辑 | 生成内容 + 原始论文 | 审核通过/修改建议 | 引用核查、事实比对 |

## 四、数据方案

### 4.1 论文数据来源

| 来源 | 类型 | 数量 | 获取方式 |
|------|------|------|---------|
| **arXiv** | AI/CS领域预印本 | 200+篇 | API批量下载（免费） |
| **Semantic Scholar** | 学术论文元数据 | 1000+条 | API获取（免费） |
| **AMiner** | 中文学术论文 | 200+篇 | 开放数据集 |
| **自建数据** | 精选核心论文 | 50-100篇 | 手动收集整理 |

### 4.2 论文数据结构

```json
{
  "paper_id": "arxiv_2024_001",
  "title": "Multi-Agent Systems: A Survey and Perspective",
  "authors": ["Wang, L.", "Chen, X.", "Zhang, Y."],
  "abstract": "This paper provides a comprehensive survey...",
  "year": 2024,
  "venue": "AAAI",
  "keywords": ["multi-agent", "cooperation", "LLM"],
  "sections": [
    {
      "title": "1. Introduction",
      "content": "Multi-agent systems have gained..."
    },
    {
      "title": "2. Related Work",
      "content": "Previous approaches include..."
    }
  ],
  "citations": 1200,
  "references": ["ref_001", "ref_002", "..."],
  "embedding_id": "vec_001"
}
```

### 4.3 知识库构建流程

```
Step 1: 数据采集（非开发者A负责）
├── 从arXiv API下载AI/Agent领域论文（500篇）
├── 从Semantic Scholar获取元数据（引用数、关键词）
└── 手动收集20篇核心论文（精读级）

Step 2: 数据清洗（非开发者A负责）
├── 去重（按标题+作者去重）
├── 格式统一（PDF → Markdown/Text）
├── 分段处理（按章节切分，每段500-1000字）
└── 元数据标注（领域、方法类型、年份）

Step 3: 向量化存储（开发者负责）
├── 使用bge-large-zh模型生成向量
├── 存入Chroma向量数据库
├── 建立元数据索引
└── 测试检索效果，调整参数

Step 4: 质量验证（全员参与）
├── 测试10个典型查询的检索准确率
├── 人工检查Top5结果的相关性
└── 根据反馈优化分块策略和检索参数
```

│  ├── 学历层次：本科生/硕士生/博士生/教师  │
│  ├── 研究方向：NLP/CV/RL/多模态/...     │

│                                         │
│  知识水平：                              │

## 六、创新点

### 创新点1：科研场景的多Agent角色分工

```
传统方案：单个大模型直接分析论文
本方案：6个Agent模拟真实科研团队的分工协作

┌─────────────────────────────────────────┐
│  模拟科研团队：                          │
│                                         │
│  协调者 → 导师（分配任务、把关方向）      │
│  检索Agent → 图书馆员（找文献）          │
│  分析Agent → 研究生（精读论文）          │
│  对比Agent → 综述作者（对比分析）        │
│  生成Agent → 写作者（撰写报告）          │
│  审核Agent → 审稿人（质量把关）          │
│                                         │
│  → 每个Agent有独立的Prompt和工具          │
│  → Agent之间有明确的分工和协作流程        │
└─────────────────────────────────────────┘
```

### 创新点2：基于用户画像的个性化文献分析

```
同一研究主题，不同用户获得完全不同的输出：

本科生 → 入门指南 + 推荐阅读路线
硕士生 → 方法对比 + 实验分析
博士生 → 研究空白 + 创新方向建议
教师   → 知识体系 + 教学素材

→ 不仅是"换个人称"，而是内容深度、
  分析角度、推荐策略都不同
```

### 创新点3：论文观点矛盾自动发现

```
对比Agent自动检测不同论文之间的观点冲突：

发现矛盾：
论文A说："集中式架构在简单任务上更高效"
论文B说："分布式架构在所有场景下都优于集中式"

→ 自动标注矛盾点
→ 引用原文佐证
→ 给出可能的解释（实验条件不同、评估指标不同等）
```

### 创新点4：Agent协同过程可视化

```
实时展示每个Agent的工作状态：
├── 当前正在做什么
├── 中间产出了什么
├── 花了多长时间
└── 最终如何汇总

→ 让"黑盒"变成"白盒"
→ 评委能直观看到多Agent的价值
```
