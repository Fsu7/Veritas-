## 一、项目概述

### 1.1 课题解读

本课题包含两个核心技术方向：

```
领域知识个性化生成          +          多智能体协同决策
       ↓                              ↓
  针对特定领域，                  多个AI Agent分工协作，
  根据用户特征，                 共同完成复杂的知识
  自动生成定制化的               分析与决策任务
  专业知识内容
```

**一句话定义**：构建一个多智能体协同系统，能够根据用户画像和需求，从领域知识库中检索、分析、生成个性化的专业知识内容，并辅助决策。

### 1.2 项目目标

| 目标层级 | 具体目标 |
|---------|---------|
| **核心目标** | 实现一个可运行的多Agent协同知识生成与决策系统 |
| **性能目标** | 知识生成准确率 > 85%，单次响应 < 5秒 |
| **创新目标** | 提出有效的多Agent协同策略和个性化生成方法 |
| **应用目标** | 在至少1个垂直领域（教育/医疗/企业知识管理）完成落地验证 |

### 1.3 预期成果

- 可运行的软件系统（Web端）
- 技术研究报告（50-80页）
- 演示视频（10-15分钟）
- 答辩PPT（20-25页）

### 3.2 多智能体角色设计

#### Agent角色定义

| Agent | 角色定位 | 核心能力 | 可用工具 |
|-------|---------|---------|---------|
| **协调者Agent** | 团队领导 | 任务分解、分配、监督、结果汇总 | 任务调度器、消息队列 |
| **检索Agent** | 信息搜集 | 从知识库中检索相关信息 | 向量检索、SQL查询、知识图谱查询 |
| **分析Agent** | 数据分析 | 分析检索结果，提取关键信息 | 统计分析、关联分析、趋势分析 |
| **对比Agent** | 对比研究 | 对比不同论文的方法、结论 | 对比引擎、矛盾发现 |
| **生成Agent** | 内容创作 | 根据分析结果生成个性化内容 | 大模型生成、模板填充 |
| **审核Agent** | 质量把关 | 检查内容的准确性和专业性 | 事实核查、一致性检查、敏感词过滤 |

#### Agent协同工作流

```
用户提问："我是大三计算机专业学生，想了解机器学习入门路线"

Step 1: 协调者Agent 接收问题
        → 分析用户画像：大三、计算机专业、零基础
        → 分解任务：
          Task1: 检索机器学习入门相关知识
          Task2: 分析适合大三学生的内容难度
          Task3: 生成个性化学习路线
          Task4: 审核内容准确性

Step 2: 检索Agent 执行Task1
        → 向量检索："机器学习入门"、"学习路线"
        → SQL查询：查找热门学习资源
        → 返回：相关文档片段 + 学习资源列表

Step 3: 分析Agent 执行Task2
        → 分析检索结果
        → 标注内容难度等级
        → 筛选适合大三学生的内容
        → 返回：结构化知识要点

Step 4: 对比Agent 执行Task3（可选，论文数≥2时激活）
        → 对比不同资源的方法和结论
        → 发现观点矛盾或差异
        → 返回：对比表格 + 差异总结

Step 5: 生成Agent 执行Task4
        → 结合用户画像 + 分析结果 + 对比结果
        → 生成个性化学习路线
        → 返回：定制化的学习建议

Step 6: 审核Agent 执行Task5
        → 检查事实准确性
        → 检查内容是否适合目标用户
        → 返回：审核通过 / 修改建议

Step 7: 协调者Agent 汇总结果
        → 整合所有Agent输出
        → 生成最终回答
        → 更新用户画像（记录兴趣偏好）

输出：一份针对该学生的个性化机器学习入门路线
```

```
用户画像模型：

│            用户画像                  │
```

## 四、创新点规划

### 4.1 核心创新点

| 创新点 | 具体内容 | 预期效果 |
|--------|---------|---------|
| **1. 基于角色分工的多Agent协同策略** | 设计协调者-检索-分析-对比-生成-审核的六角色协同模式，提出任务分解与结果融合算法 | 相比单Agent，复杂问题回答准确率提升15%+ |
| **2. 动态个性化生成机制** | 基于用户画像的实时内容定制，包括难度适配、风格匹配、推荐增强 | 不同用户同一问题获得差异化回答 |
| **3. 知识图谱增强的RAG** | 向量检索 + 知识图谱推理的混合检索策略，提高知识召回准确率 | 知识检索准确率 > 90% |
| **4. Agent协同过程可视化** | 实时展示各Agent的工作状态、中间结果、决策过程 | 提高系统可解释性和可信度 |

3. **与发榜单位契合**：科大讯飞本身就是教育AI领域的头部企业
4. **演示效果好**：可以现场演示不同学生获得不同学习方案
