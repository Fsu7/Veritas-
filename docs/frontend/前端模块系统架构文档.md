## 1 文档概述

### 1.1 编写目的

本文档详细定义科研文献智能助手系统中前端5大模块的系统架构，包括页面设计、组件规范、状态管理、API通信、Agent可视化等内容，为前端开发提供完整的设计蓝图。

### 1.2 适用范围

本文档覆盖前端全部5个子模块：

| 模块编号 | 模块名称 | 核心职责 |
|---------|---------|---------|
| F1.1 | 用户界面模块 | 注册/登录、画像设置/编辑、历史记录 |
| F1.2 | 论文检索模块 | 主题输入、智能检索、结果展示、筛选排序 |
| F1.3 | 论文分析模块 | 论文详情、智能分析、通俗解释、多论文对比 |
| F1.4 | 综述生成模块 | 综述生成、个性化输出、报告导出、引用溯源 |
| F1.5 | Agent可视化模块 | 工作流展示、状态监控、中间结果、耗时统计 |

### 1.3 技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| **Vue3** | 3.4+ | 前端框架（Composition API + `<script setup>`） |
| **TypeScript** | 5.0+ | 类型安全 |
| **Vite** | 5.0+ | 构建工具 |
| **Element Plus** | 2.5+ | UI组件库 |
| **ECharts** | 5.4+ | Agent工作流可视化、耗时统计图表 |
| **Pinia** | 2.1+ | 状态管理 |
| **Vue Router** | 4.2+ | 路由管理 |
| **Axios** | 1.6+ | HTTP客户端 |
| **markdown-it** | 14.0+ | Markdown渲染（综述内容） |

### 2.2 数据流转

```
用户操作
    │
    ▼
View（页面）→ Component（组件）
    │ 触发Action
    ▼
Pinia Store
    │ 调用API
    ▼
Axios / SSE
    │ HTTP请求 / SSE连接
    ▼
Java后端 API
    │
    ▼
响应数据
    │
    ▼
Store更新State → 组件响应式更新 → 视图刷新
```

### 2.3 前端与后端通信架构

```
┌──────────────────────────────────────────┐
│            前端（Vue3）                    │
│                                          │
│  Axios HTTP Client                       │
│  ├── 请求拦截：注入JWT Token             │
│  ├── 响应拦截：统一错误处理              │
│  └── 基础URL：/api                       │
│                                          │
│  SSE Client                              │
│  ├── EventSource连接                     │
│  ├── 接收Agent状态更新                   │
│  └── 自动重连（3秒间隔）                 │
└──────────────────┬───────────────────────┘
                   │
                   │ Nginx反向代理
                   │ /api/* → java-backend:8080
                   ▼
┌──────────────────────────────────────────┐
│         Nginx (端口80)                    │
│  ├── / → 静态文件                        │
│  └── /api/* → proxy_pass java-backend    │
└──────────────────┬───────────────────────┘
                   │
                   ▼
┌──────────────────────────────────────────┐
│      Java后端 (Spring Boot, 端口8080)     │
│  ├── /api/users/*     → 用户管理          │
│  ├── /api/papers/*    → 论文管理          │
│  ├── /api/sessions/*  → 会话管理          │
│  ├── /api/analysis/*  → 分析服务          │
│  └── /api/analysis/*/agent-stream → SSE   │
└──────────────────────────────────────────┘
```

## 8 Agent可视化模块（F1.5）

### 8.1 模块概述

实时展示多Agent协同工作流程图和运行状态，是系统的核心差异化展示特性。基于ECharts Graph图实现。

### 8.2 AgentFlowChart组件

```typescript
// components/agent/AgentFlowChart.vue
<script setup lang="ts">
import { ref, computed, watch, onMounted, onUnmounted } from 'vue'
import * as echarts from 'echarts'
import { useAgentStore } from '@/stores/agentStore'

const agentStore = useAgentStore()
const chartRef = ref<HTMLDivElement>()
let chartInstance: echarts.ECharts | null = null

// Agent节点定义
const agentNodes = [
  { name: 'coordinator', label: '协调者', x: 400, y: 50 },
  { name: 'retriever', label: '检索', x: 200, y: 150 },
  { name: 'analyzer', label: '分析', x: 400, y: 150 },
  { name: 'comparer', label: '对比', x: 600, y: 150 },
  { name: 'generator', label: '生成', x: 400, y: 250 },
  { name: 'reviewer', label: '审核', x: 400, y: 350 }
]

// Agent连线定义
const agentLinks = [
  { source: 'coordinator', target: 'retriever' },
  { source: 'coordinator', target: 'analyzer' },
  { source: 'retriever', target: 'analyzer' },
  { source: 'analyzer', target: 'comparer' },
  { source: 'analyzer', target: 'generator' },
  { source: 'comparer', target: 'generator' },
  { source: 'generator', target: 'reviewer' },
  { source: 'reviewer', target: 'coordinator' }  // 审核反馈
]

// 根据Agent状态计算节点颜色
function getNodeColor(agentName: string): string {
  const state = agentStore.getAgentState(agentName)
  switch (state?.status) {
    case 'running': return '#409EFF'    // 蓝色（执行中，含脉冲动画）
    case 'completed': return '#67C23A'  // 绿色（已完成）
    case 'failed': return '#F56C6C'     // 红色（失败）
    default: return '#C0C4CC'           // 灰色（等待中）
  }
}

// ECharts配置
const chartOption = computed(() => ({
  tooltip: {
    formatter: (params: any) => {
      const state = agentStore.getAgentState(params.name)
      return `${params.data.label}<br/>
              状态：${state?.status || '等待中'}<br/>
              ${state?.intermediateResult || ''}`
    }
  },
  series: [{
    type: 'graph',
    layout: 'none',
    symbolSize: 60,
    roam: false,
    label: {
      show: true,
      fontSize: 14,
      formatter: (params: any) => params.data.label
    },
    edgeSymbol: ['none', 'arrow'],
    edgeSymbolSize: [4, 10],
    data: agentNodes.map(node => ({
      name: node.name,
      label: node.label,
      x: node.x,
      y: node.y,
      itemStyle: {
        color: getNodeColor(node.name),
        borderColor: getNodeColor(node.name),
        borderWidth: 2
      }
    })),
    links: agentLinks,
    lineStyle: {
      color: '#909399',
      width: 2,
      curveness: 0.1
    }
  }]
}))

// 监听Agent状态变化，更新图表
watch(() => agentStore.agentStates, () => {
  chartInstance?.setOption(chartOption.value)
}, { deep: true })

onMounted(() => {
  if (chartRef.value) {
    chartInstance = echarts.init(chartRef.value)
    chartInstance.setOption(chartOption.value)
  }
})

onUnmounted(() => {
```

```typescript
      favorites.value.push(paperId)
    }
```

```typescript
  (config) => {
    const userStore = useUserStore()
    if (userStore.token) {
      config.headers.Authorization = `Bearer ${userStore.token}`
    }
    return config
  },
  (error) => Promise.reject(error)
)

// 响应拦截器：统一错误处理
http.interceptors.response.use(
  (response) => {
    const data = response.data as ApiResponse<any>
    if (data.code === 200) {
      return data.data
    }
    // 业务错误
    ElMessage.error(data.message || '请求失败')
    return Promise.reject(new Error(data.message))
  },
  (error) => {
    if (error.response?.status === 401) {
      // Token过期或无效
      const userStore = useUserStore()
      userStore.logout()
      router.push('/login')
      ElMessage.error('登录已过期，请重新登录')
    } else if (error.response?.status === 403) {
      ElMessage.error('无权限访问')
    } else if (error.response?.status === 404) {
      ElMessage.error('请求的资源不存在')
    } else if (error.code === 'ECONNABORTED') {
      ElMessage.error('请求超时，请稍后重试')
    } else {
      ElMessage.error(error.response?.data?.message || '网络错误')
    }
    return Promise.reject(error)
  }
```

## 13 SSE实时通信

### 13.1 useSSE组合函数

> **架构重构说明**：SSE 已从旧的 `connectAgentStream` 直接调用模式重构为 `useSSE` composable + `watch` 模式。页面通过 `watch(analysisId, ...)` 监听 analysisId 变化自动建立/断开 SSE 连接，`connectAgentStream` 方法已移除。此模式确保组件卸载时自动断开连接，且 analysisId 变化时自动重连。

```typescript
// composables/useSSE.ts
import { ref, onUnmounted } from 'vue'
import { useAgentStore } from '@/stores/agentStore'

export function useSSE(analysisId: Ref<string | null>) {
  const agentStore = useAgentStore()
  const eventSource = ref<EventSource | null>(null)
  const isConnected = ref(false)
  const reconnectAttempts = ref(0)
  const maxReconnectAttempts = 5

  function connect() {
    if (!analysisId.value) return

    const url = `/api/analysis/${analysisId.value}/agent-stream`
    eventSource.value = new EventSource(url)
    isConnected.value = true
    reconnectAttempts.value = 0

    // 监听Agent状态更新事件
    eventSource.value.addEventListener('agent_state_update', (event) => {
      const data = JSON.parse(event.data)
      agentStore.updateAgentState(data.agentName, {
        status: data.status,
        progress: data.progress,
        intermediateResult: data.intermediateResult,
        durationMs: data.durationMs
      })
    })

    // 监听分析完成事件
    eventSource.value.addEventListener('analysis_completed', (event) => {
      const data = JSON.parse(event.data)
      // 更新最终状态
      disconnect()
    })

    // 连接错误处理
    eventSource.value.onerror = () => {
      isConnected.value = false
      eventSource.value?.close()

      // 自动重连
      if (reconnectAttempts.value < maxReconnectAttempts) {
        reconnectAttempts.value++
        setTimeout(connect, 3000)
      }
    }
  }

  function disconnect() {
    eventSource.value?.close()
    eventSource.value = null
    isConnected.value = false
  }

  onUnmounted(() => {
    disconnect()
  })

  return {
    isConnected,
    connect,
    disconnect
  }
}
```

## 17 部署架构

### 17.1 构建与部署

```bash
# 构建生产版本
npm run build
# 输出目录：dist/

# Nginx配置
server {
    listen 80;
    server_name localhost;

    # 前端静态文件
    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;    # SPA路由支持
    }

    # API代理到Java后端
    location /api/ {
        proxy_pass http://java-backend:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;

        # SSE支持
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 300s;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|svg|ico)$ {
        root /usr/share/nginx/html;
        expires 1d;
        add_header Cache-Control "public, immutable";
    }
}
```
